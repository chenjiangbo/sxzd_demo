# 担保业务简报生成模块 - 使用说明

## 📋 功能概述

本模块用于自动生成"业务一部担保业务简报"，支持以下功能：

- ✅ 从本地 JSON/CSV 文件读取 12 张业务表格数据
- ✅ 时间期间选择（上半年/下半年/自定义）
- ✅ Word 模板自动填充（单个指标 + 表格数据）
- ✅ 生成并下载 Word 文档
- ✅ 左侧菜单式表格浏览界面

## 📁 数据文件说明

### 1. 表格数据文件
**路径**: `data/brief-tables.json`

**结构**:
```json
{
  "metadata": {
    "title": "业务一部担保业务简报",
    "version": "1.0",
    "lastUpdated": "2026-03-31"
  },
  "tables": [
    {
      "id": "table_1",
      "name": "担保业务总体情况表",
      "category": "总体概况",
      "data": [
        {"指标": "在保余额", "数值": "125.6 亿元", "较年初增长": "8.5%"}
      ]
    }
    // ...共 12 张表格
  ]
}
```

**12 张表格清单**:
1. 担保业务总体情况表
2. 分产品类型担保情况表
3. 分行业担保情况表
4. 分地区担保情况表
5. 合作银行担保业务统计表
6. 代偿情况统计表
7. 追偿情况统计表
8. 准备金提取情况表
9. 经营效益指标表
10. 新增担保业务情况表
11. 重点客户担保情况表
12. 再担保分险情况表

### 2. 占位符映射文件
**路径**: `data/placeholder-mapping.json`

**作用**: 定义 Word 模板中的占位符与实际数据的对应关系

**结构**:

    "table_placeholders": {
      "{{table_overview}}": "table_1",
      "{{table_product_structure}}": "table_2",
      "...": "..."

## 📝 Word 模板制作指南

### 第一步：创建模板文件

1. **打开 Microsoft Word 或 WPS**
2. **设计页面布局**
   - 设置页眉页脚
   - 选择字体（推荐：标题黑体，正文宋体）
   - 调整页边距

3. **编写模板内容**

**示例结构**:
```
                    {{report_title}}
                    
报告期间：{{report_period}}

一、总体概况

截至{{report_period}}末，在保余额{{total_balance}}，
较年初增长{{total_balance_growth}}；
在保户数{{total_accounts}}，增长{{total_accounts_growth}}。

{{table_overview}}

二、业务结构分析

1. 产品结构
{{table_product_structure}}

2. 行业结构
{{table_industry_structure}}

三、风险控制

当期代偿金额{{compensation_amount}}，
代偿率{{compensation_rate}}。

{{table_compensation}}

...（更多内容）
```

### 第二步：插入占位符

**占位符语法**: `{{key}}`

**两类占位符**:

1. **单个指标占位符** (24 个)
   ```
   {{report_title}}           - 报告标题
   {{report_period}}          - 报告期间
   {{total_balance}}          - 在保余额
   {{total_balance_growth}}   - 较年初增长
   {{compensation_amount}}    - 代偿金额
   {{compensation_rate}}      - 代偿率
   ...等等
   ```

2. **表格占位符** (12 个)
   ```
   {{table_overview}}              - 总体情况表
   {{table_product_structure}}     - 产品结构表
   {{table_industry_structure}}    - 行业结构表
   ...等等
   ```

### 第三步：保存模板

- **格式要求**: `.docx` (不能是 `.doc`)
- **文件名**: `brief-template.docx`
- **存放位置**: `templates/` 目录

### 任务 2: 完善 API 实现
修改 `app/api/generate-brief/route.ts` 中的 `fillWordTemplate` 函数：

```typescript
import Docxtemplater from 'docxtemplater';
import PizZip from 'pizzip';

async function fillWordTemplate(
  templatePath: string,
  indicators: Record<string, string>,
  tables: Record<string, any[][]>
): Promise<Buffer> {
  // 1. 读取模板文件
  const templateContent = await fs.readFile(templatePath, 'binary');
  
  // 2. 创建 zip 实例
  const zip = new PizZip(templateContent);
  
  // 3. 创建 docxtemplater 实例
  const doc = new Docxtemplater(zip, {
    paragraphLoop: true,
    linebreaks: true,
  });
  
  // 4. 准备数据（包括表格 HTML）
  const tableHTMLs: Record<string, string> = {};
  Object.entries(tables).forEach(([tableId, tableData]) => {
    tableHTMLs[tableId] = `
      <table border="1">
        ${tableData.map(row => `
          <tr>${row.map(cell => `<td>${cell}</td>`).join('')}</tr>
        `).join('')}
      </table>
    `;
  });
  
  // 5. 合并所有数据
  const allData = {
    ...indicators,
    ...tableHTMLs,
  };
  
  // 6. 渲染文档
  doc.render(allData);
  
  // 7. 生成 buffer
  const buf = doc.getZip().generate({
    type: 'nodebuffer',
    compression: 'DEFLATE',
  });
  
  return buf;
}
```

## ❓ 常见问题

### Q1: 如何替换成真实的 12 张表格数据？
**A**: 编辑 `data/brief-tables.json`，修改 `tables` 数组中的 `data` 字段，填入真实数据。

### Q2: CSV 格式支持吗？
**A**: 当前版本使用 JSON。如需 CSV，可以：
1. 将 CSV 转换为 JSON（使用 Papa Parse 库）
2. 或者在 API 层读取 CSV 并转换为 JSON

### Q3: Word 模板的表格占位符如何精确定位？
**A**: docxtemplater 支持多种表格插入方式：
- 直接替换文本占位符为 HTML 表格
- 使用 docxtemplater 的循环语法在模板中定义表格结构
- 参考 docxtemplater 官方文档的 table 示例

### Q4: 如何添加更多指标或表格？
**A**: 
1. 在 `data/placeholder-mapping.json` 中添加新的占位符映射
2. 在 `data/brief-tables.json` 中添加新的表格数据
3. 在 Word 模板中插入对应的占位符
