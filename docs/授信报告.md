## 一、先把边界定死
### 这次 demo 做什么
只做后半段：

**读表 → 读制度 → 归纳授信结论 → 生报告 → OA 按钮**

### 这次 demo 不做什么
不做这些：

+ 不做国担平台实时取数
+ 不做复杂重算引擎
+ 不做规则配置后台
+ 不做 OA 真对接
+ 不做通用 skill / agent 配置系统

### 这次 demo 的输入
你现在就用这 3 个输入：

1. **授信情况统计表 Excel**
2. **授信管理办法 PDF**
3. **历史/样例报告 docx**

### 这次 demo 的输出
输出 3 个东西就够：

1. **AI 授信结论摘要**
2. **正式报告草稿**
3. **提交 OA 按钮**

---

## 二、为什么这样设计
**AI 能不能把现成表里的结果讲明白、归纳清楚、写成正式报告，并把需要人判断的地方单独拎出来。**

---

## 三、你最关心的那个点：人工确认到底设计成什么
工确认只做两件事：

### 1. 确认“口径标签”
例如某家机构：

+ 按申请额度配置
+ 按政策目标值配置
+ 按测算值配置
+ 新机构按政策目标值配置

### 2. 填一条备注
例如：

+ “按历史口径确认”
+ “需业务负责人复核”
+ “本机构为特殊机构，沿用样稿结论”

**不要给用户一个数值输入框，让他去改授信额度。**  
那样这个 demo 会立刻变成“半成品算表工具”，反而不好讲。

你这次应该让用户确认的是：

**“这个结论怎么解释、这类机构归到哪一类、报告能不能发。”**

不是“我现场改一堆数字”。

## 四、完整流程，就 5 步
### 第 1 步：打开“年度授信任务”
用户进入页面后，系统直接展示本次任务概览：

+ 41 家合作机构
+ 申请总额 489.3 亿元
+ 拟授信总额 451.3 亿元
+ 银担分险额度 375.6 亿元
+ 非银担分险额度 75.7 亿元
+ 国企额度 2.13 亿元

这里先让领导一眼看到结果盘子。

---

### 第 2 步：AI 自动读表，生成“授信结论摘要”
系统读取 Excel 里的现成字段，自动归纳出：

+ 哪些机构属于“按申请额度配置”
+ 哪些机构属于“按政策目标值配置”
+ 哪些机构属于“按测算值配置”
+ 哪家是“新加入体系，按政策目标值配置”

这一步不是让 AI 去“重新算”，而是**把表里的结果翻译成业务能看懂的话**。  
2025 年报告样稿本身已经给了你现成分组逻辑和话术。

---

### 第 3 步：AI 结合制度，解释“规则 / 口径”
这是你新加的重点功能，我建议做成全局右侧抽屉，不要单独做配置后台。

用户点任意字段，比如：

+ 额度系数
+ 再担保增长率
+ 非分险额度
+ 分险额度
+ 机构评价
+ 评级系数

右侧就弹出：

**A. 制度条文解释**  
**B. 2025 年实际执行口径**  
**C. 这次表里对应的是哪一列**

这个设计非常重要，因为你这里的“规则”其实有两层：

#### 第一层：制度条文
比如制度明确写了：

+ 授信额度 = 上年度再担保规模 ×（1 + 本年度再担保增长率）× 额度系数
+ 额度系数由 8 项指标加权组成
+ 产品额度分为银担分险和非银担分险
+ 非银担分险额度按本年度合作政策确定
+ 银担分险额度 = 总授信额度 - 非银担分险额度

#### 第二层：2025 年实际执行口径
比如样稿里写的是：

+ 2025 年非银担分险额度按 2024 年备案规模压降 10%
+ 14 家机构的国企担保贷款额度按 2023 年该类业务规模的 40% 给
+ 最终授信按“申请额度 / 政策目标值 / 测算值 / 新机构口径”四类处理

**这两个不能混。**

所以制度助手里一定要有两个标签：

+ **制度条文**
+ **本年执行口径**

这样用户就不会误以为“压降 10%”是制度写死的，而会明白这是 2025 年的年度执行口径。

---

### 第 4 步：人工确认
这一页只放 3 个区域：

#### ① AI 归纳结论
一句话摘要，比如：

+ 本年授信以年度目标导向为主，结合申请额度和时间节点完成率，形成四类配置结论。

#### ② 待确认事项
只列少量关键项：

+ 循环担保是否按新机构处理
+ 个别机构分组是否沿用样稿口径
+ 非分险压降 10% 口径是否沿用
+ 国企额度口径是否沿用

#### ③ 操作按钮
+ 确认生成报告
+ 修改备注后生成
+ 导出摘要
+ 提交 OA

注意：  
这里仍然**不要让用户改具体数值**。  
顶多允许改“口径标签”和“备注”。

---

### 第 5 步：生成正式报告
这一页就是成文页。

AI 根据：

+ Excel 里的结果
+ 制度里的标准说法
+ 你给的历史报告样稿语气

自动生成：  
**《关于 2025 年度合作担保机构再担保业务授信的报告》**

报告结构就沿用你现有样稿：

1. 综合授信情况
2. 产品分项额度设定
3. 授信额度运用

生成后下面放两个按钮：

+ **复制 / 导出报告**
+ **提交 OA**

OA 这里你说得对，做个按钮走样子就行。

## 五、页面怎么做，最简 4 页就够
## 页面 1：年度授信任务页
目的：看全局

模块：

+ 顶部 5 张统计卡
+ 中间机构列表

按钮：

+ 读取统计表
+ 生成授信结论

---

## 页面 2：AI 授信结论页
目的：看“为什么这么给”

模块：

+ 左侧：四类机构分组
+ 中间：每类金额汇总
+ 右侧：待确认事项

这里最关键的是把 2025 样稿里的四类逻辑直接可视化：  
申请额类、政策目标值类、测算值类、新机构类。

---

## 页面 3：制度口径助手
目的：随点随看制度，不翻 PDF

建议做成右侧抽屉，所有页面都能打开。

默认展示 6 个快捷卡片：

+ 授信额度
+ 8 项指标
+ 评级 A/B/C 折算
+ 非分险和分险额度
+ 授信资料
+ 授信审批

这些内容制度里都有依据。

用户点击任何表头，也能自动打开对应说明。

这个页面的价值非常高，因为它把“读制度”变成“查口径”。

---

## 页面 4：报告生成与 OA
目的：出最终成果

模块：

+ 左侧：报告预览
+ 右侧：本次采用口径摘要
+ 底部：OA 按钮

按钮：

+ 重新生成
+ 导出 Word
+ 提交 OA

---

## 六、制度助手里具体放什么内容
这个你可以直接照着做。

### 快捷问答示例
用户点一下就能看：

**1. 额度系数是什么意思？**  
展示 8 项指标和权重。

**2. 评级 A/B/C 怎么换算？**  
A=100 分，B=80 分，C=60 分。

**3. 非分险额度怎么来的？**  
先展示制度条文“按本年度合作政策确定”，再展示 2025 年执行口径“在 2024 基础上压降 10%”。

**4. 为什么不是所有机构都按测算值授信？**  
展示样稿里的四类授信决策逻辑。

**5. 授信资料要报什么？**  
展示制度第二十一条材料清单。

**6. 后面审批怎么走？**  
展示 A/B 角、风控法务部、总经理办公会这条线。

## 八、建议  
就抓住 3 个亮点：
**第一，AI 会读表并归纳授信结论。**  
**第二，AI 会把制度变成可查询的口径助手。**  
**第三，AI 会把结果写成正式报告并挂一个 OA 按钮。**

这 3 个点已经足够像一个能替业务一部岗位干活的 demo 了。

如果你愿意，我下一条就直接给你出：  
**4 个页面的详细原型文案 + 每个区域显示什么字段 + 按钮文案。**
